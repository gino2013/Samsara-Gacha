# Samsara Gacha — 現行設計文件

> 記錄 `index.html` 目前實作的設計決策，供後續調整時對照。單一檔案、無建置流程，這份文件本身也是唯一的設計說明。

## 產品概念

「投胎轉生機」：轉蛋機制包裝的輪迴主題小遊戲。抽到的每個「結果」是一個國家（或次分類，如「美國・矽谷科技菁英圈」），分級呈現，作為對全球生活水準差異的科普／宣傳素材。免責聲明明確聲明分級不代表對該國家或人民的評價。

## 視覺設計系統

### 色彩（CSS variables，`:root`）

| Token | 值 | 用途 |
|---|---|---|
| `--void` | `#05060B` | 背景底色 |
| `--panel` / `--panel2` | `#12141F` / `#1B1E2E` | 卡片、面板背景 |
| `--line` | `rgba(255,255,255,0.08)` | 邊框 |
| `--text` / `--muted` | `#EDEEF5` / `#8A8FA3` | 主文字／次要文字 |
| `--ur1/2/3` | `#FFD866` `#FF6EC7` `#6EE7FF` | UR 專屬三色漸層（金／粉／青） |
| `--ssr` `#FFB627` / `--sr` `#8B7FFF` / `--r` `#4EA1FF` / `--n` `#6B7280` | 各稀有度單色 |
| `--num-score/happy/gdp/cost` | 琥珀／青／藍／粉 | 說明文字中數值 inline highlight |

背景疊加兩層徑向漸層 + 固定星點紋理（`.stars`，纯 CSS `radial-gradient` 堆疊，非圖片）。

### 字型

- `Orbitron`：標題、按鈕、稀有度徽章（科技／電玩感)
- `Noto Sans TC`：內文（中文閱讀）
- `JetBrains Mono`：數字、統計、機率（等寬強調「數據感」)

三者皆由 Google Fonts CDN 載入。

### 元件

- **稀有度卡片**（`.card`）：依 tier 套用對應邊框色 + 發光陰影，UR 額外有 `sweep` 掃光動畫與更強雙色光暈。
- **十連結果格**（`.mini` × 10）：進場用 `miniIn` 逐格淡入位移動畫。
- **粒子爆炸**（`.burst`）：抽卡時依 tier 產生 18～28 個彩色粒子（UR 更多、更炫)。
- **3D 地球儀**（`#globeCanvas`）：手刻 `<canvas>` 等距投影（見下方「地球儀渲染」)。
- **文字自動平衡**（`balanceText`）：用 `canvas.measureText` 量測寬度，在標點處插入換行，避免卡片標題/敘述斷行難看（`text-wrap: balance` 的手動 polyfill)。

## 抽卡機制

### 稀有度權重與收錄國家數

| Tier | 權重 | 機率 | 收錄筆數 |
|---|---|---|---|
| UR | 1 | 1% | 4 |
| SSR | 6 | 6% | 13 |
| SR | 13 | 13% | 25 |
| R | 12 | 12% | 25 |
| N | 68 | 68% | 135 |

機率為權重佔總和（100）的比例，抽獎用線性掃描 `rollTier()` 實作（`Math.random() * totalWeight` 後依序扣減)。

### 保底（pity）

- 追蹤 `sincePity`，每抽 +1；抽到 SSR 或 UR 時歸零。
- 達到第 10 抽時強制 `forcePityBreak`：只在 UR / SSR 兩池之間依權重（1:6）加權抽取，保證出貨。
- UI 顯示「距離保底還有 N 抽」（`pityCount`）。

### 單抽 / 十連

- 十連是呼叫 `rollOne()` 十次，並挑出十抽中稀有度最高者作為主卡展示結果（`rank` 映射 UR=0…N=4，取最小值)。

## 評分公式（決定 tier 分類的來源資料）

國家排序依「綜合評分」計算，公式與說明同時寫死在 footer 文案與每筆資料的 `reason` 字串中：

```
綜合評分 = 32.2 + 5.28 × 快樂指數（0–10） + 0.21 × 人均GDP（千美元） − 0.36 × 生活成本指數
```

- 快樂指數：World Happiness Report，0–10 分。
- 人均GDP：IMF／World Bank／UN 名目數據，美元。
- 生活成本指數：Numbeo，以美國紐約市 = 100 為基準。

這個公式**只用來決定資料前置作業時的排序／分級**，前端程式碼本身不重新計算— `TIERS` 裡每個國家的 `reason` 字串已經內嵌算好的分數與三項原始數值（例如 `"綜合評分106.7分（快樂指數7.80・人均GDP $288,002・生活成本指數75.3）"`），前端只用正則 `formatReason()` 解析後上色顯示，不做計算。

## 地球儀渲染

- 純手刻正交（orthographic-like）投影：`project(lat, lon, yaw, tilt, radius)` 將經緯度轉成 2D 座標 + 深度值（`depth`)，用於：
  - 決定描邊粗細／透明度（近大遠小、背面淡出)
  - 大陸輪廓（`WORLD_LAND` 硬編碼多邊形陣列）依深度加陰影
- 支援閒置自轉（`idleSpin`）與點擊结果後的動畫飛行（`animateToLocation`：先轉 yaw 到目標經度、緩動用 `easeOutCubic`，再放大 zoom)。
- 定位後在畫布上疊一個 HTML `.pin` 元素（非畫在 canvas 上），靠 `updatePinPosition` 換算 canvas 座標到 DOM 座標。

## 頁面結構（由上到下）

1. Header（標題、標語)
2. Stats 列（總抽數 + 各 tier 累積次數)
3. 主抽卡卡片 + 數值圖例（快樂指數/GDP/生活成本說明)
4. 單抽／十連按鈕 + 保底倒數
5. 十連結果格（十連時顯示)
6. 地球儀面板（定位／重置按鈕 + 地理資訊卡)
7. 歷史紀錄列（可點擊重新定位)
8. Footer（免責聲明 + 機率 + 評分公式)

## 技術棧

純 vanilla HTML/CSS/JS 單檔（`index.html`，~1150 行），無建置工具、無依賴、無後端。動畫用 CSS keyframes + `requestAnimationFrame`（地球儀轉動/縮放)。資料（國家清單、座標、評分文案）直接寫死在 `TIERS` 常數中。
