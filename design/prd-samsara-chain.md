# PRD v2 — 多世輪迴（Samsara Chain）擴充規劃

> 狀態：**Phase 1（MVP，FR-1～FR-5）已實作**，其餘 Phase 2／3 仍是規劃中。本檔案是團隊群組討論（與 Lucian、阿寶）整理出的 feature backlog，收斂進 `design/` 供排期參考。對照現況實作請見 [`DESIGN.md`](./DESIGN.md)。

## 一句話說明

把現有「單次投胎」重新定義成「一整條輪迴鏈」：玩家從一個隨機的過去起點開始，一世一世投胎下去，每世的國家平均壽命決定下一世的年份，最終無論中途抽到什麼，最後一世都會收斂在 **2026 年**——也就是玩家的當下。

## 目標

1. 從「開盲盒」升級成「走完一生又一生，直到今生」的敘事體驗。
2. 用起始年份 + 各國/時代平均壽命，讓多世投胎自然收斂到 2026。
3. 用有限的投胎次數取代無限抽卡，做出稀缺感與儀式感。
4. 敏感題材（國家分級、歷史身分）處理原則要文件化，不再逐案討論。
5. 文本內容盡量用 AI 生成，避免手工窮舉「時代 × 身分 × 國家」組合。

## 非目標（本階段不做）

- 完整古代～現代時間軸（先做近現代版，古代/中世紀留待未來 DLC）。
- 玩家自由選擇初始國籍（見下方「與現有功能的衝突」）。
- 多人對戰／社群排行榜。

## FR-6 與現有滑動選國籍功能的關係（已決策：選項 A）

FR-6 的原意是「起點完全隨機，不開放手動選國籍」，理由是玩家自選高稀有度國家會不尊重機率本質。而我們在同一天已實作了「一進站可以滑動選自己現世國籍」的進場畫面（`#onboardOverlay`，見 DESIGN.md）。

**決策：兩者不衝突，滑動選國籍保留，但重新定義用途。** 玩家滑動選的不是「投胎起點」，而是 FR-5 的「終局收斂目標」——輪迴鏈的第一世依然完全隨機，只是最後一世（2026）會收斂到玩家選的這個國家，呼應「無論投胎去哪，最終還是要回到自己身上」的主題。

現有 `#onboardOverlay` 的文案（「你這一世的國籍是？」／「現世國籍」）已經符合這個定位，不需要改字；等 FR-1～FR-5（多世輪迴鏈）實作時，`selectedHome`（見 index.html 的 `HOME_COUNTRY_KEY`）直接接進 FR-5 的收斂邏輯即可，是既有資料，不用重做選擇 UI。

## 詳細規格摘要

### FR-1～FR-5 實作說明（Phase 1 MVP，已上線）

不新增一套平行 UI，直接把既有的「十連」按鈕重新定義成一次完整的輪迴鏈，最大化重用既有的 `rollOne()` / 保底 / 統計邏輯：

1. 點擊「十連 · 投胎十次」→ 彈出 `#yearPickerOverlay`（沿用 `.onboard` 樣式），用原生 `<input type="range" min="1776" max="2010">` 選起始西元年（FR-2）。
2. 確認後執行 `runSamsaraChain(startYear)`：跑原本一模一樣的 `rollOne()` 十次（**保底機制完全沿用，不改**——10 世＝原本的 10 抽保底週期，開放問題 2 已決策），只是額外幫每個結果算一個「西元年」：
   - 第 1 世 = 玩家選的起始年。
   - 第 2～9 世 = 上一世年份 + `TIERS[tier].lifespan × (0.8~1.2 隨機)`。`lifespan` 掛在**等級**上（UR 85 / SSR 78 / SR 70 / R 60 / N 45），不是逐國維護，換取資料量最小（FR-3，PRD 原文允許「國家/等級」二選一）。
   - 第 10 世（最後一世）**強制** `year = 2026`，不改動抽卡結果本身的國家/稀有度，只覆蓋顯示的年份（FR-5）。
3. 世數上限固定為 10（FR-4，開放問題 1 已決策：先做固定世數，不做動態反推），跟十連的既有次數自然對齊，沒有另外做計數 UI。
4. 十連結果格（`.mini`）每張卡片新增「第 N 世・西元 XXXX」小字。
5. 跑完後 1.2 秒，彈出 `#chainEndOverlay` 結語畫面，文案採用「這一世，你還記得嗎？你一直想成為的人。你在這裡不會沒有原因，一切都是最好的選擇。」，並附上起點/終點國家的一句摘要。

「單抽・轉世一次」按鈕維持原樣，不參與輪迴鏈邏輯，避免把兩種心智模型混在一起。

### FR-6～FR-11：仍是規劃中，尚未實作（見下方 Feature List 與詳細規格）

### FR-7 分時代身分池
以「時代區間 + 身分清單 + 事件標籤」為資料單位，方便未來加入大航海時代等 DLC。中國、印度等可有獨立身分池分支，避免套用西方中心敘事。

### FR-8 AI 動態生成時代文本
年份＋國家＋身分即時（或預先批次生成快取）產出敘事文案。需要 prompt 範本 + 種子文本人工審核，避免聳動或片面政治表述。

### FR-9 週目／輪迴佇列
投胎次數用盡 → 「本週目已結束」畫面 → 可開啟新一輪（回到 FR-2）。可選擇記錄歷史週目次數。

### FR-10 敏感分級處理準則
獨立文件說明：資料來源與計算方式透明化（沿用 DESIGN.md 的評分公式）、爭議國家維持「純數據驅動、不主觀加權」立場、台灣等條目的人工維護原則。

### FR-11 大國內部分級
先為 1–2 個代表性大國（如中國）設計「城市線級」子分級資料結構原型，抽到後可再細分顯示（如「上海（一線）」）。

## 開發順序

1. **Phase 1（MVP）**：FR-1 → FR-2 → FR-3 → FR-4 → FR-5 —— **已完成**（先做出「多世收斂到 2026」的核心體驗，單一時代身分，不做歷史 DLC）
2. **Phase 2（敘事深化）**：FR-6（決策落地）→ FR-7（現代版身分池）→ FR-8（AI 生成文本）
3. **Phase 3（長期擴充）**：FR-9 → FR-10（文件類工作，可提前到 Phase 1）→ FR-11
