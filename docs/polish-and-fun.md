# 質感與好玩度升級提案

> 目標：讓「投胎轉生機」從「有趣的機率玩具」升級成「有儀式感、想分享、玩完會留下情緒的小體驗」。
> 依「投入 / 回報」排序，✅ = 本次已實作（2026-07-13）。

## A. 抽卡儀式感（特效）— 回報最高

抽卡遊戲的爽感 80% 來自「揭示前的期待 + 揭示瞬間的爆發」。目前是按下按鈕 → 結果直接出現，中間沒有張力。

| 項目 | 做法 | 狀態 |
|---|---|---|
| 蓄力階段 | 按下抽卡後卡片先收縮、增亮 ~0.5s，再彈出結果（期待感） | ✅ |
| 全螢幕粒子引擎 | 一層固定 `<canvas>`，火花（additive blending）、彩帶（旋轉下落）、衝擊波環，依稀有度給量：UR 150+ 火花 + 60 彩帶 + 雙衝擊波 | ✅ |
| 螢幕閃光 | 揭示瞬間全螢幕稀有度色 radial 閃光，0.7s 淡出（SSR/UR） | ✅ |
| 螢幕震動 | UR/SSR 揭示時整個版面 shake 0.55s | ✅ |
| 卡片全息傾斜 | 滑鼠移動時卡片 3D 傾斜 + 跟隨游標的彩虹 holo 光層（UR/SSR），手感像實體閃卡 | ✅ |
| 星空閃爍 + 極光 | 星點層加呼吸動畫；背景加一層緩慢漂移的模糊極光 | ✅ |
| 音效 | 不加素材檔，用 WebAudio 合成：蓄力上升音、揭示 chime（稀有度越高和弦越華麗）、UR 專屬長音。約 60 行 | 待做 |
| 十連逐張揭示 | 輪迴鏈的每一世逐張翻開（可跳過），而不是一次全出 | 待做 |

## B. UI 質感

| 項目 | 做法 | 狀態 |
|---|---|---|
| 保底進度條 | 純文字「還有 N 抽」改為漸層進度條 + 文字 | ✅ |
| 按鈕掃光 | hover 時斜向白光掃過（CSS ::after） | ✅ |
| 卡片 holo 光層 | 見上，CSS 變數 + `mix-blend-mode:screen` | ✅ |
| 行動端優化 | 粒子數在 `matchMedia(pointer:coarse)` 下減半；tilt 只在有 hover 的裝置啟用 | 待做 |
| prefers-reduced-motion | 關閉裝飾動畫（極光、星空、環） | ✅ |
| 深色以外主題 | 不建議做，深色是這個題材的正確答案（YAGNI） | 不做 |

## C. 玩法深度（讓人玩第二次的理由）

| 項目 | 做法 | 預估 |
|---|---|---|
| 圖鑑收集 | 已抽過的國家點亮（localStorage 一個 Set），顯示「收集 37/202」，未收集以剪影顯示。收集慾是 gacha 的核心留存 | ~120 行 |
| 分享卡生成 | 用 canvas 把「輪迴鏈總結」畫成一張圖（起點年、世數、最高稀有度、金句），`toBlob` + Web Share API / 下載。病毒傳播的關鍵 | ~150 行 |
| 輪迴鏈統計 | 回顧頁加總：總共活了 N 年、育有 N 名子女、最常見的職業、最短的一世。數字會自己講故事 | ~40 行 |
| 成就系統 | 「三世皆 N」「一鏈雙 UR」「活過 90 歲」等 10~15 個成就，toast 提示 + 成就牆 | ~150 行 |
| 每日命運籤 | 以日期為亂數種子的每日一抽，「今日份的輪迴」，給回訪理由 | ~30 行 |
| 前世今生對照 | 已有現世國籍資料，可加「這一世 vs 你的現世」快樂指數/GDP 對比條 | ~60 行 |

## D. 文本（感人且合理）

| 項目 | 做法 | 狀態 |
|---|---|---|
| 生平敘事化 | 「出身：X／職業：Y／死因：Z」標籤式改寫成連貫句子：「出身商旅世家…這一世，你以『行商』度日。24歲那年成了家…61歲，離開人間——瘟疫。」 | ✅ |
| 結尾文案打磨 | 輪迴總結、追問、安息頁逐句重寫：具體意象 > 抽象安慰，「帶著前世們留給你的、你不記得的溫柔」 | ✅ |
| 今生進行式 | 回顧頁最後一世（現世）補一句「這一世還在書寫中——接下來的故事，換你自己寫。」 | ✅ |
| 死因語感統一 | `data.js` 各死因池混雜動詞句（戰死沙場）與名詞（缺血性心臟病），敘事模板已用「離開人間——X」相容兩者；長期可統一為名詞短語 | 部分 |
| 生平事件池擴充 | 每個職業 category 加 1~2 件「人生中段事件」（例：行商「曾在異鄉大病一場，被素不相識的人收留」），讓每世多一個記憶點 | 待做（data.js） |
