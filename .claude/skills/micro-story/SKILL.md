# 極短敘事（micro-story）寫作技巧

研究來源：六字小說（"For sale: baby shoes, never worn."）的分析文獻、中文微型小說三要素（構思新奇、情節完整、結尾出人意料）。
核心結論：**觸動人的從來不是字數多，是留白的位置。**

## 五條鐵律

### 1. 留白 — 最痛的那句話不寫出來
讀者自己補完的情緒，比作者寫明的強十倍。
- ❌ 「她走了，你難過了很久，因為她像你的親人一樣。」
- ✅ 「她走的那天，你破例讓全府歇了一日。」
悲傷不寫「悲傷」二字，寫悲傷讓人做出的事。

### 2. 具體物件承載情感 — 不用抽象詞
baby shoes 之所以成立，是因為「鞋」是實物。抽象詞（愛、遺憾、思念）零重量；
物件（一支筆、一盞油燈、一副碗筷、一支不成調的歌）有重量。
每條人生線必須有一個貫穿的**錨**：一件物、一個人、一句話。

### 3. 埋線與呼應 — setup 在童年，payoff 在晚年
單句再美，六句互不相干就只是六句格言。關聯感的來源：
- 童年埋下錨（老僕婦哼的歌）
- 中段讓錨發展、變化（你補了她二十年沒漲的月錢）
- 晚年錨回來，但**換了一個形狀**（新來的丫頭哼起同一支歌 — 是她教的）
呼應不是重複：回來的東西必須帶著時間的痕跡。

### 4. 結尾轉一下 — 最後一句必須讓前面的話變重
微型小說三要素的「結尾出人意料」不是反轉劇，是**視角一翻**：
- 「搶匪裡有人認出你，喝住了所有人：這家的帳，撕過我娘的。」
善行的 payoff 從受助者口中說出、身分揭曉延後到最後一刻、
死亡之後才打開的箱子 — 都是同一招：把鑰匙放在最後一句。

### 5. 每個字打兩份工 — 砍到不能再砍
一句話 15-35 字。刪掉：解釋句（「那一刻你懂了…」）、總結句（「這句話你記了一輩子」）、
形容詞堆疊。動作與對白留下，感想全刪 — 感想是讀者的工作。
- ❌ 「你在深宅大院裡長大，僕人們喊你小主人。你最喜歡躲上後院的老樹，看牆外市集的熱鬧 - 那是你唯一看得到的外面。」（55字）
- ✅ 「你爬上後院的老樹，第一次看見牆外。」（17字）

## Samsara 專用結構

LIFE_CHAPTERS 每個職業類別 = **12 條人生線（threads）**，不是 6 個獨立句池。
同一條線貫穿 childhood → youth → early → peak → midlife → twilight 六階段，
六個 stage 陣列的**同一個 index 屬於同一條線**，抽取時整世固定用同一 index。

一條合格的線 = 一篇拆成六格的極短篇：
```
childhood: 埋錨（具體物件/人物登場）
youth:     錨與主角建立關係
early:     第一次考驗（錨受威脅或被選擇）
peak:      錨開花（但不點破）
midlife:   失去或交棒
twilight:  錨換一個形狀回來（payoff + 轉）
```

## 硬性約束

- 每句 15-35 字，絕不超過 45。
- 全域共用池禁用文化專屬詞 — 完整清單見 `test_shared_pool.js` 的 SINO_ONLY
  （提親/科舉/衙門/祠堂/算盤/功名…）。寫完必跑 `node test_shared_pool.js`。
- 六個 stage 陣列長度必須相等（thread 對齊依賴 index 平行）。
- 一世可能早夭：線被截斷在任何一格都要能獨立成立，childhood 單獨一句也得是完整畫面。

## 檢查清單（每條線寫完自問）

1. 錨是實物/實人嗎？能一句話說出「這條線關於什麼」嗎？
2. 把六句連起來讀，像不像一篇 100 字小說？
3. 最後一句拿掉，前五句是否明顯變輕？（是 → 結尾合格）
4. 有沒有任何一句在「解釋情緒」？（有 → 刪）
5. 隨便抽掉中間兩句（模擬早夭/階段跳過），剩下的還通嗎？
